# Changelog

## [3.3.1]

### Changed
- Replaced `onLinkClickHandler` closure with `PolarAppDelegate` protocol in `PolarApp.initialize()`
- `PolarApp.initialize()` now accepts a `delegate: PolarAppDelegate?` parameter instead of a callback closure
- Updated `QuickIntegration` notification methods to use completion handler-based UNUserNotificationCenterDelegate API (Obj-C compatible) instead of async/await

### Added
- New `PolarAppDelegate` protocol with two delegate methods:
  - `polarApp(_:didClickLink:data:error:)` — called when a deep link is clicked
  - `polarApp(_:didReceiveNotification:data:)` — called when a push notification is received
- `PushClient` now holds a weak reference to `InternalPolarApp` to forward notification events to the delegate

### Fixed
- Improved memory safety with `[weak self]` captures in async dispatch blocks
